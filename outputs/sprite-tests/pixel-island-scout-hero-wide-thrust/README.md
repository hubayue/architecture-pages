# 主角横向长突刺宽帧边界测试

## 结论

本次测试通过了素材导出层验证：横向长突刺并不需要被改成紧凑近身攻击，也不必直接改为手工美术处理。工作流新增宽攻击格输出后，已经能够保留长突刺动作本身，并输出透明精灵表、单帧、GIF、提示词和带 Pivot 的元数据。

## 本次补齐的工作流能力

- 普通动作仍使用 `128 x 128` 正方形帧格。
- 长突刺攻击使用独立 `224 x 128` 宽帧格。
- 元数据记录固定运行时 Pivot：图像坐标 `112, 119`，归一化坐标 `0.5, 0.9296875`。
- 宽动作处理时以原生成格中的本体中心保持运行时原点，不再因武器横向延展而把角色本体自动挤偏。
- 同时检查生成源格与最终宽帧是否出现裁切或越界。

## 验收结果

- 四帧均无源格越界。
- 四帧均无最终宽帧裁切。
- 精灵表尺寸为 `448 x 256`，每帧尺寸为 `224 x 128`。
- 长突刺最大轮廓宽度为 `150 px`，可完整容纳在宽帧中。
- 平均高度相对 `idle` 偏差为 `-5.79%`；其中起手和收势帧仍为 `108 px`，突刺帧因身体前倾而自然降低。

## 边界与后续

这次证明的是 AI 素材生成与宽帧交付能力已可实现该动作。尚未验证的是实际引擎中的 Pivot 坐标转换、动作切换播放、攻击命中框与角色碰撞框配置。

正式生产路径不应默认改动作规避宽度问题，而应根据动作设计选择帧规格。下一项验证应是将标准动作与宽帧突刺动作放入 Unity 或 Godot 的最小预览场景中播放。

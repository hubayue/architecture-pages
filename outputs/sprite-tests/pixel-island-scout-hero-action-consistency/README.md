# 主角跨动作一致性测试 - 最终整理结果

## 最终结论

本次测试最终通过：`idle / walk / attack / hurt` 可保持同一主角的身份、像素风格、尺寸级别与脚底对齐。

通过所依赖的边界条件是：`attack` 采用紧凑的近身下劈姿势，而不是横向长突刺。当前固定单元格能够稳定容纳近身攻击，不能在不缩小角色或越界的前提下容纳原横向长突刺。

## 最终素材

最终可用动作均位于 `accepted/`，每个动作保留项目标准交付项：

- `idle/`：作为一致性基准的待机动作。
- `walk/`：通过的行走动作。
- `attack/`：通过的紧凑近身攻击动作。
- `hurt/`：通过的受击动作。

每个动作目录包含：

- `raw-sheet.png`
- `raw-sheet-clean.png`
- `sheet-transparent.png`
- 四张单帧 PNG
- `animation.gif`
- `prompt-used.txt`
- `pipeline-meta.json`

## 验收结果

- 角色身份：通过。棕色层叠发型、贝壳发饰、青绿色眼睛、珊瑚围巾、短披风标记、卷轴筒与弯木杖保持一致。
- 风格：通过。各动作保持同一套精致热带像素风处理。
- 尺寸：通过。`walk` 相对 `idle` 为 `0.00%`，`attack` 为 `-2.31%`，`hurt` 为 `-3.94%`，均在 `15%` 容差内。
- 对齐：通过。所有动作的脚底基准线均为 `119 px`。
- 切帧边界：通过。最终动作均无跨格或贴边失败帧。

## 两轮测试沉淀

第一次测试发现：一个外观非常接近基准角色的横向突刺 `attack` 会跨入相邻单元格；为消除越界而直接重新生成的替换动作，又出现发型和武器细节漂移。因此第一次测试未能同时满足身份与边界要求。

第二次测试的解决方式是：强化身份固定条件，同时把攻击动作约束为紧凑、身体内聚、无独立特效的近身下劈。这样既保住角色识别特征，也通过固定格的边界检查。

可复用经验：

- 主角多动作生成应始终以已通过的 `idle` 为身份基准。
- 尺寸和边界约束不能只在产出后检查，应直接写入动作设计要求。
- 长武器横向突刺属于规格问题，不宜反复依靠提示词压缩；若必须使用，应提供更宽攻击格或单独的攻击导出规格。
- 中间失败样本适合转化为文字结论与数据记录，最终交付目录仅保留通过版本。
